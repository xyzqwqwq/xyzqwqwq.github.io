maven生命周期

maven包含三个标准的生命周期：default、clean以及site

maven执行某一个生命周期中的命令时，并不是单独运行命令对应的阶段，而是会从头开始运行。

default生命周期 描述 内置生命周期绑定（插件:目标）
validate 验证项目是否正确以及所有必要的信息均可用
initialize 初始化构建状态，例如设置属性或创建目录
generate-sources 生成任何要包含在编译中的源代码
process-sources 处理源代码
generate-resources 生成资源，使其包含在classpath路径下
process-resources 将资源复制并处理到目标目录中，以备打包 resources:resources
compile 编译项目源代码 compile:compile
process-classes 对编译后生成的文件进行处理
generate-test-sources 生成测试源代码
process-test-sources 处理测试源代码
generate-test-resources 生成测试资源
process-test-resources 处理测试资源 resources:testResources
test-compile 将测试源代码编译到测试目录中 compile:testCompile
process-test-classes 处理测试编译生成的文件
test 使用单元测试框架运行测试 surefire:test
prepare-package 在打包前执行操作 jar:jar或war:war
package 获取编译后的文件，将其打包为可分发格式
pre-integration-test 部署到集成测试前执行的操作
integration-test 处理程序包并将其部署到集成测试环境
post-integration-test 部署到集成测试后执行的操作
verify 验证包是否有效并符合质量标准
install 将包发布到本地仓库 install:install
deploy 将包发布到远程仓库 deploy:deploy
clean生命周期 描述 内置生命周期绑定（插件:目标）
pre-clean 在项目清理前执行操作
clean 项目清理，删除默认生命周期生成的文件 clean:clean
post-clean 在项目清理后执行操作
site生命周期 描述
pre-site 在项目文档生成前执行操作
site 生成项目文档
post-site 生成项目文档后执行操作，并在部署前执行操作
site-deploy 将文档部署到指定服务器

插件配置

在maven中有两种插件，即build和report:

  • build插件在构建期间执行，并在<build/>元素中配置
  • report插件在网站生成期间执行，在<reporting/>元素中配置
